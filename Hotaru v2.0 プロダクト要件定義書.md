# Hotaru v2.0 プロダクト要件定義書

**バージョン:** 2.0
**更新日:** 2026年3月5日

---

## 1. プロジェクト概要

### 1.1 背景と課題
旅先で、誰かに伝えたい場所に出会ったことがあるだろうか。地元の人しか知らない小さな豆腐屋。100年続く銭湯。夕暮れに染まる名もない路地——。そういう場所への愛は、GoogleマップのレビューにもInstagramにも書けない。評価でも「映え」でもないからだ。でも誰かに伝えたい。次に来た旅人に、そっと渡したい。

現代の旅行サービスはすべて「情報」か「映え」に最適化されている。しかし、旅人が本当に持ち帰りたいのは、その場所への「愛」だ。Hotaruはその愛だけを集める。

### 1.2 ソリューション
**旅の始まりに「旅をはじめる」ボタンを押す。** 旅の記録がスタートし、その間に感じた愛だけが「今回の旅」として記録される。写真を撮り、3つの感情ボタンから1つを選ぶ。言葉は一切要らない。旅の終わりに「旅をおわる」ボタンを押すと、その旅でどんな感情を抱いたかのサマリーが表示される。

ユーザーの愛が集まるほど、その場所が地図の上で蛍のように光る。口コミでも評価でもない。人の愛が場所の本質を決める地図が生まれる。

### 1.3 3つの感情カテゴリー
| 感情 | 光の色 | どんな愛か |
| :--- | :--- | :--- |
| 🟠 たのしい | オレンジ | 笑った、美味しかった、また来たい。心が弾んだ場所への愛。 |
| 🔵 うつくしい | ブルー | 景色、光、空気。言葉にならない美しさに出会った場所への愛。 |
| 🩷 のこしたい | ピンク | なくなってほしくない、ずっとあってほしい。場所への祈りに近い愛。 |

### 1.4 ターゲットユーザー
- 旅を「消費」ではなく「深く味わう」ことを大切にする25〜40代
- 地域の文化・暮らし・風景に愛着を感じる人
- 口コミを書くほどではないが「誰かに伝えたい」と思う人

### 1.5 プロダクトビジョン
**「人の愛が、地図を書き換える。」**

### 1.6 Hotaruが「しないこと」
| しないこと | 理由 |
| :--- | :--- |
| 言葉でレビューさせる | 愛は言葉にならない。言葉にした瞬間に口コミになる。 |
| 星・点数で評価させる | 愛に優劣はない。評価は比較を生むだけ。 |
| ランキングを作る | 愛の多さで場所の価値を決めたくない。 |
| 既存の写真から投稿させる | 現地で感じた愛だけを地図に灯す。地図の信頼性を守る。 |

## 2. 機能要件

### 2.1 コア機能（MVP必須）

**① 旅のセッション管理（Trip Session）** `[v2.0 NEW]`
- **旅の開始:** ホーム画面の「旅をはじめる」ボタンを押すと、新しい旅セッションが開始される。
- **記録中:** 旅のセッション中は、画面上部に「旅の記録中...」と表示される。
- **旅の終了:** 「旅をおわる」ボタンを押すとセッションが終了し、後述の「旅の感情サマリー」画面が表示される。

**② 愛を残す（Love Capture）**
- **旅のセッション中のみ**カメラを起動して写真を撮れる。
- 3つの感情ボタン（🟠たのしい / 🔵うつくしい / 🩷のこしたい）から1つ選ぶ。
- 自動付与：Geolocation APIで現在地・時刻を記録し、進行中の`trip_id`と紐付ける。

**③ 蛍の地図（Hotaru Map）**
- 日本地図上に、全ての旅から集まった愛の場所が蛍のように光って浮かぶ。
- フィルター機能はv1と同様。

**④ 旅の感情サマリー（Trip Summary）** `[v2.0 NEW]`
- 「旅をおわる」ボタンを押した直後に表示されるモーダル画面。
- **表示内容:**
    - 「今回の旅の軌跡」として、訪れた場所の数と総移動距離（簡易）。
    - 「今回の旅の感情」として、3つの感情（🟠🔵🩷）がそれぞれ何回記録されたかを視覚的に表示（例：色付きのバーグラフやサークルグラフ）。
    - 「おつかれさまでした。あなたの愛は、次の旅人を照らします。」というメッセージ。

**⑤ 自分の愛のカードアルバム**
- **旅ごとのグループ化:** 自分が愛を残したカードが「2026年3月 京都の旅」「2026年5月 沖縄の旅」のように、旅単位で自動的にグループ化されて表示される。
- 各旅のヘッダーには、その旅の感情サマリー（ミニ版）が表示される。
- カード自体の仕様はv1と同様。

### 2.2 非機能要件
- v1と同様（オフライン対応、PWA対応など）。

## 4. 開発スケジュール（48時間ハッカソン） `[v2.0 UPDATE]`

**Day 1**
- **0〜4h:** 環境構築、カメラ＋感情ボタンUI
- **4〜8h:** **旅セッション（Start/Stop）実装、`trips`テーブル作成、`loves`テーブルへの`trip_id`紐付け**
- **8〜12h:** 蛍の地図実装（Leaflet + Three.js）

**Day 2**
- **0〜4h:** **カードアルバムの旅ごとグループ化表示**
- **4〜8h:** **旅の感情サマリー画面実装（まずはシンプルな数字表示でOK）**
- **8〜12h:** PWA通知、デザイン仕上げ、デモ準備

### MVP判定基準（この5つが動けば合格）
1.  「旅をはじめる」を押すと記録が開始され、「旅をおわる」で終了する。
2.  旅の間に記録した愛が、地図に蛍として光る。
3.  カードアルバムで、記録が旅ごとにまとまって表示される。
4.  「旅をおわる」を押した後に、感情の集計結果（数字だけでも可）が表示される。
5.  近くに蛍がある場所でPush通知が届く。
